# 🍏 Food Analyzer (Client-Server Java Application)

Food Analyzer е многонишково клиент-сървър приложение, написано на Java. То позволява на потребителите да търсят детайлна хранителна информация за различни продукти (калории, протеини, мазнини, въглехидрати и др.) чрез интеграция с официалното API на **USDA (U.S. Department of Agriculture)**.

Приложението поддържа търсене по име на продукт, търсене по баркод, както и иновативен *fallback* механизъм за извличане и разчитане на баркодове директно от графични изображения (снимки).

---

## ✨ Ключови функционалности

* **Многонишковост (Multi-threading):** Сървърът може да обслужва множество клиенти едновременно (Concurrent Clients), благодарение на `ExecutorService` и thread-safe колекции (`CopyOnWriteArrayList`).
* **Интеграция с външно API:** Комуникация с базите данни на USDA чрез HTTP заявки (`HttpService`) и десериализация на JSON данни в Java обекти.
* **Интелигентно кеширане (Caching):** Веднъж потърсена, информацията за даден продукт се запазва локално (`CacheService`), за да се избегнат излишни мрежови заявки и да се оптимизира скоростта.
* **Баркод скенер & Fallback:** Възможност за търсене на продукт по текстов баркод. Ако кодът не бъде открит, клиентът може автоматично да разчете баркод от подадено изображение (`BarcodeUtils`).
* **Устойчивост на грешки (Resilience):** Имплементирани са защити срещу прекъсване на мрежата, изключване на сървъра по време на работа (Graceful Shutdown) и валидация на потребителския вход.

---

## 🏗 Архитектура

Проектът е разделен на два основни модула:
1.  **Сървърна част (`FoodAnalyzerServer`):** Приема TCP връзки от клиентите, комуникира с USDA API, кешира резултатите и връща обработената информация.
2.  **Клиентска част (`FoodAnalyzerConsole`):** Интерактивен терминал (CLI), който чете команди от потребителя, изпраща ги към сървъра чрез `NetworkClient` и визуализира резултатите. Включва логика за обработка на изображения.

## 🚀 Инсталация и стартиране

### Предварителни изисквания
* **USDA API Key:** Необходим е валиден ключ от [FoodData Central](https://fdc.nal.usda.gov/api-key-signup.html).

### 1. Настройка на API Ключ
Преди да стартирате сървъра, трябва да зададете вашия API ключ като системна променлива (Environment Variable):

**За Linux / macOS:**
`export USDA_API_KEY="твой_истински_ключ_тук"`

**За Windows (Command Prompt):**
`set USDA_API_KEY="твой_истински_ключ_тук"`

*(Забележка: Ако използвате IntelliJ IDEA, добавете променливата в полето `Environment variables` на конфигурацията на `ServerApp`.)*

### 2. Стартиране
1.  Първо стартирайте **сървъра**: Изпълнете класа `ServerApp.java`. Сървърът ще тръгне и ще слуша за връзки (по подразбиране на порт `18080`).
2.  След това стартирайте **клиента(ите)**: Изпълнете класа `ClientApp.java`. Можете да стартирате множество клиенти паралелно.

## 💻 Потребителски команди (CLI)

След като клиентът се свърже успешно със сървъра, можете да използвате следните команди:

| Команда | Описание | Пример |
| :--- | :--- | :--- |
| `get-food <name>` | Търси храна по име и връща нейните хранителни стойности. | `get-food apple` |
| `get-food-by-barcode --code=<code>` | Търси храна по конкретен баркод (FDC ID / GTIN). | `get-food-by-barcode --code=12345` |
| `get-food-by-barcode --code=<code> --img=<path>` | Търси по код. Ако не бъде намерен, опитва да прочете баркода от посочената снимка. | `get-food-by-barcode --code=000 --img=apple_barcode.png` |
| `get-food-by-barcode --img=<path>` | Извлича баркод от снимката и търси продукта директно по него. | `get-food-by-barcode --img="C:\images\barcode.jpg"` |
| `disconnect` | Прекратява връзката със сървъра и затваря клиента. | `disconnect` |
| `exit` | Аналогично на `disconnect`. | `exit` |

*На сървърната конзола можете да въведете командата `stop`, за да изключите безопасно сървъра и да разкачите всички активни клиенти.*

## 🧪 Тестване

Проектът включва изчерпателен набор от Unit тестове, покриващи бизнес логиката, мрежовата комуникация и изключенията.
* Използвани библиотеки: **JUnit** и **Mockito**.
* Критични компоненти като `NetworkClient`, `FoodAnalyzerConsole`, `ClientHandler` и `CacheService` са тествани чрез Dependency Injection (Mocking) за независими и бързи тестове, без нужда от реални мрежови сокети или активен интернет.
